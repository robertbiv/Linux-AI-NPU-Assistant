## 2025-04-16 - [Fix TOCTOU and file descriptor leaks in temporary file creation]
**Vulnerability:** A Time-of-Check to Time-of-Use (TOCTOU) vulnerability where `tempfile.mkstemp` was used securely, but its file descriptor was immediately closed, and the file was reopened via its path string during a network download. Also, if the network download failed before reopening, the unclosed file descriptor would leak.
**Learning:** Security features like `mkstemp` return file descriptors that guarantee exclusive, atomic access to the newly created file. Closing this descriptor and referring to the file by path reintroduces the race condition `mkstemp` was meant to prevent, allowing symlink or file-replacement attacks. Moreover, raw integers returned by `mkstemp` must be wrapped in `os.fdopen` *before* any fallible operations (like network requests) to ensure cleanup on error.
**Prevention:** Always immediately wrap the file descriptor from `tempfile.mkstemp` using `with os.fdopen(fd, ...)` before performing network or slow operations. Never close the descriptor just to reopen the file by its string path.
